## Known Constraints & Gotchas

### Adapter Design Decisions
- **IntelX 2-phase search**: POST search → poll up to 4 times → GET terminate. Recovers gracefully if base URL fails; tries 2nd URL fallback (2.intelx.io then free.intelx.io). _MAX_RESULTS=10, _MAX_POLLS=4.
- **AbstractPhone & VirusTotal**: use the standard `_net.http_get()` pattern, no Tor (authenticated by key).
- **Shodan**: returns subdomains as Tier-3 domain_hit units (triggers further domain recon if a domain is re-seeded).
- **All keyed adapters**: health_check() returns `bool(_api_key())` → unhealthy when key blank → degrades to unavailable gracefully.
